并发容器：ConcurrentHashMap

发表于2022-12-04|更新于2025-10-24|并发编程

Java 7 版本的 ConcurrentHashMap


在 ConcurrentHashMap 内部进行了 Segment 分段，Segment 继承了 ReentrantLock，可以理解为一把锁，各个 Segment 之间都是相互独立上锁的，互不影响。相比于之前的 Hashtable 每次操作都需要把整个对象锁住而言，大大提高了并发效率。
每个 Segment 的底层数据结构与 HashMap 类似，仍然是数组和链表组成的拉链法结构，最大并发个数就是 Segment 的个数，默认是 16。

Java 8 版本的 ConcurrentHashMap

当链表长度大于某一个阈值（默认为 8），且同时满足一定的容量要求的时候，ConcurrentHashMap 便会把这个链表从链表的形式转化为红黑树的形式，目的是进一步提高它的查找性能。
（默认是链表长度达到 8 就转成红黑树，而当长度降到 6 就转换回去，这体现了时间和空间平衡的思想）

  1. 锁粒度更细，理想情况下 table 数组元素的个数就是其支持并发的最大个数，并发度比之前有提高；
  2. 放弃了 Segment 的设计，采用 Node + CAS + synchronized 保证线程安全；
  3. 先使用拉链法，在链表长度超过一定阈值时，将链表转换为红黑树，来提高查找效率。
